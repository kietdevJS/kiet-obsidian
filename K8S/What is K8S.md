- Traditional: Deploy multiple app on a single server and os -> resource hog, hard to config
- Virtualize: Hard to config (ip config for each vms,..) and each VM requires resources to run the Os alone
- Container: Each app is separated on each container, flexible resource config for each app
- K8S: Automatic scaling, error fix,...

K8S is an automation tool, config deployment of applications
